# Complex Dialog Sample

This sample creates a complex conversation with dialogs.

This copy is based on Microsoft's `43.complex-dialog` sample (BotBuilder-Samples)
and has been extended across three assignments:

- **Week 5** added a "capabilities" command/welcome message and reworked the idle
  message handler so out-of-scope input no longer gets swallowed as the start of
  the name/age dialog.
- **Week 6** integrated Google's **Gemini API** as a cloud AI-as-a-service: any
  idle input that isn't a recognized command (`start`/`restart`/`begin`,
  `help`/`capabilities`) is now answered by Gemini instead of a static message.
- **Week 7** reshaped the Gemini integration into **Sage**, a supportive
  listening companion, applying conversational-agent design methods (persona,
  CUX fallbacks, human-centered safety). See "What Week 7 changed" below.

## What Week 7 changed (Sage)

Week 6 sent the user's raw text to Gemini with no instruction, persona, or
memory. Week 7 modified that single AI call and the message handler to turn the
bot into a purpose-built listening agent:

- **Persona + personality system prompt** (`helpers/ai_helper.py`): Gemini now
  receives a system instruction that gives it a stable identity ("You are
  Sage… not a licensed therapist") kept separate from its tone (warm, brief,
  reflective), following the persona/personality distinction from the design
  literature.
- **A fail-safe crisis path** (`helpers/safety.py`): a deterministic keyword
  screen runs on the user's input *before* any model call, so crisis resources
  (US 988/911) surface even if the API is down. Matching input latches a crisis
  state that only clears on an explicit acknowledgement ("I'm safe", "I'll
  call"). A model-level safety instruction is a second layer for phrasings the
  keyword screen misses (defense in depth).
- **Capped fallbacks**: a first failed Gemini call retries; a second
  *consecutive* failure escalates to a hand-off message (CUX Guide's
  two-attempt limit). A successful call resets the streak.
- **Bounded conversation memory**: the last several turns are passed with each
  call so Sage can follow context across the conversation.
- **Removed the original name/age/company-review dialog**: the structured
  multi-step dialog no longer fit a listening-companion persona, so the dialog
  files were deleted and the bot is now built around `bots/listening_bot.py`
  (with `bots/welcome_bot.py` greeting new users).

## Bot Capabilities

- Greets new users as **Sage** on join, and on `help` / `capabilities` /
  `what can you do` explains it is a supportive listening companion and not a
  licensed therapist.
- Holds a reflective, context-aware listening conversation for any input,
  generated by Gemini with Sage's persona and tone (see `helpers/ai_helper.py`).
- Detects direct crisis language and responds with crisis resources, staying in
  the crisis path until the user acknowledges (see `helpers/safety.py`).
- Degrades gracefully when the AI service fails, retrying once and then escalating
  to a hand-off message.
- Remembers recent conversation turns for the duration of the conversation.

## To try this sample

- Clone the repository
    ```bash
    git clone https://github.com/manishfsb/AI4HCI/Week7.git
    ```
- In a terminal, navigate to the `Week7/43.complex-dialog` folder
- Activate your desired virtual environment
- Create a `.env` file in the repository root containing your Gemini API key:
  ```
  GEMINI_API_KEY=your-gemini-api-key-here
  ```
  (Get a free key from [Google AI Studio](https://aistudio.google.com). Without
  this, the bot still runs, but falls back to a "not configured" message for
  any input Gemini would otherwise answer.)
- In the terminal, type `pip install -r requirements.txt`
- Run your bot with `python app.py`

## Testing the bot using Bot Framework Emulator

[Bot Framework Emulator](https://github.com/microsoft/botframework-emulator) is a desktop application that allows bot developers to test and debug their bots on localhost or running remotely through a tunnel.

- Install the latest Bot Framework Emulator from [here](https://github.com/Microsoft/BotFramework-Emulator/releases)

### Connect to the bot using Bot Framework Emulator

- Launch Bot Framework Emulator
- File -> Open Bot
- Enter a Bot URL of `http://localhost:3978/api/messages`
